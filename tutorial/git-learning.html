What is Git?

Git is a distributed version control system that tracks changes in source code during software development. It allows multiple developers to work on the same project efficiently.

Key Features:

  • Distributed version control
  • Branching and merging
  • Data integrity and security
  • Speed and performance
  • Offline capabilities

Git vs Other VCS:

  • Distributed - Every clone is a full backup
  • Snapshots - Stores data as snapshots, not differences
  • Integrity - Everything is checksummed
  • Three States - Modified, Staged, Committed

Git Installation & Setup

Installation:

# Windows (using Chocolatey)
choco install git

# macOS (using Homebrew)
brew install git

# Ubuntu/Debian
sudo apt-get install git

# CentOS/RHEL
sudo yum install git

Initial Configuration:

# Set user name and email
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

# Set default editor
git config --global core.editor "code --wait"

# Check configuration
git config --list
git config user.name

Basic Git Commands

Repository Initialization:

# Initialize new repository
git init

# Clone existing repository
git clone https://github.com/user/repo.git
git clone https://github.com/user/repo.git my-project

Basic Workflow:

# Check repository status
git status

# Add files to staging area
git add filename.txt
git add .                    # Add all files
git add *.js                 # Add all JavaScript files

# Commit changes
git commit -m "Commit message"
git commit -am "Add and commit in one step"

# View commit history
git log
git log --oneline
git log --graph --oneline --all

File Operations:

# Remove files
git rm filename.txt
git rm --cached filename.txt  # Remove from staging, keep in working directory

# Move/rename files
git mv oldname.txt newname.txt

# View differences
git diff                      # Working directory vs staging
git diff --staged            # Staging vs last commit
git diff HEAD~1              # Current vs previous commit

Git Branching

Branches allow you to work on different features simultaneously without affecting the main codebase.

Branch Operations:

# List branches
git branch                   # Local branches
git branch -r               # Remote branches
git branch -a               # All branches

# Create new branch
git branch feature-branch
git checkout -b feature-branch  # Create and switch
git switch -c feature-branch    # Modern syntax

# Switch branches
git checkout main
git switch main             # Modern syntax

# Delete branch
git branch -d feature-branch    # Safe delete
git branch -D feature-branch    # Force delete

Branch Management:

# Rename branch
git branch -m old-name new-name

# Track remote branch
git branch --set-upstream-to=origin/main main

# Show branch information
git show-branch
git branch -v               # Show last commit on each branch

Merging and Rebasing

Merging:

# Merge branch into current branch
git merge feature-branch

# Merge with no fast-forward (creates merge commit)
git merge --no-ff feature-branch

# Abort merge in case of conflicts
git merge --abort

Rebasing:

# Rebase current branch onto main
git rebase main

# Interactive rebase (last 3 commits)
git rebase -i HEAD~3

# Continue rebase after resolving conflicts
git rebase --continue

# Abort rebase
git rebase --abort

Conflict Resolution:

# When conflicts occur:
# 1. Edit conflicted files
# 2. Remove conflict markers (<<<<<<<, =======, >>>>>>>)
# 3. Add resolved files
git add conflicted-file.txt
# 4. Complete merge/rebase
git commit  # For merge
git rebase --continue  # For rebase

Remote Repositories

Remote Management:

# Add remote repository
git remote add origin https://github.com/user/repo.git

# List remotes
git remote -v

# Change remote URL
git remote set-url origin https://github.com/user/new-repo.git

# Remove remote
git remote remove origin

Synchronization:

# Fetch changes from remote
git fetch origin
git fetch --all

# Pull changes (fetch + merge)
git pull origin main
git pull --rebase origin main  # Pull with rebase

# Push changes
git push origin main
git push -u origin main        # Set upstream
git push --all                 # Push all branches
git push --tags               # Push tags

Collaboration Workflows

Feature Branch Workflow:

# 1. Create feature branch
git checkout -b feature/user-authentication

# 2. Work on feature
git add .
git commit -m "Add user login functionality"

# 3. Push feature branch
git push -u origin feature/user-authentication

# 4. Create Pull Request (on GitHub/GitLab)
# 5. After review, merge to main
# 6. Delete feature branch
git branch -d feature/user-authentication
git push origin --delete feature/user-authentication

Forking Workflow:

# 1. Fork repository on GitHub
# 2. Clone your fork
git clone https://github.com/yourusername/repo.git

# 3. Add upstream remote
git remote add upstream https://github.com/originaluser/repo.git

# 4. Keep fork updated
git fetch upstream
git checkout main
git merge upstream/main
git push origin main

Advanced Git Commands

Stashing:

# Stash current changes
git stash
git stash push -m "Work in progress"

# List stashes
git stash list

# Apply stash
git stash apply
git stash apply stash@{2}

# Pop stash (apply and remove)
git stash pop

# Drop stash
git stash drop stash@{1}

Reset and Revert:

# Reset to previous commit
git reset --soft HEAD~1     # Keep changes staged
git reset --mixed HEAD~1    # Keep changes unstaged (default)
git reset --hard HEAD~1     # Discard all changes

# Revert commit (creates new commit)
git revert HEAD
git revert abc123

Cherry-pick:

# Apply specific commit to current branch
git cherry-pick abc123
git cherry-pick abc123..def456  # Range of commits

Tags:

# Create tag
git tag v1.0.0
git tag -a v1.0.0 -m "Version 1.0.0"

# List tags
git tag
git tag -l "v1.*"

# Push tags
git push origin v1.0.0
git push origin --tags

# Delete tag
git tag -d v1.0.0
git push origin --delete v1.0.0

Git Workflows

1. Centralized Workflow:

  • Single main branch
  • All developers commit to main
  • Simple but limited

2. Feature Branch Workflow:

  • Feature development in separate branches
  • Merge to main via pull requests
  • Code review before merging

3. Gitflow Workflow:

# Main branches: main, develop
# Supporting branches: feature, release, hotfix

# Start feature
git flow feature start new-feature

# Finish feature
git flow feature finish new-feature

# Start release
git flow release start 1.0.0

# Finish release
git flow release finish 1.0.0

4. GitHub Flow:

  • Anything in main is deployable
  • Create descriptive branch names
  • Commit early and often
  • Open pull request for discussion
  • Deploy after review
  • Merge after deployment

Git Best Practices

Commit Messages:

# Good commit message format:
# <type>(<scope>): <subject>
# 
# <body>
# 
# <footer>

feat(auth): add user login functionality

Implement JWT-based authentication system with:
- Login endpoint
- Token validation middleware
- Password hashing

Closes #123

General Best Practices:

  • Commit often - Small, logical commits
  • Write clear messages - Explain what and why
  • Use branches - Keep main stable
  • Review code - Use pull requests
  • Keep history clean - Use rebase for feature branches
  • Use .gitignore - Exclude unnecessary files
  • Tag releases - Mark important versions
  • Backup regularly - Push to remote frequently

Git Interview Questions

1. What is the difference between git merge and git rebase?

Answer: Merge creates a new commit combining two branches, preserving history. Rebase replays commits on top of another branch, creating linear history.

2. What is git stash and when to use it?

Answer: Git stash temporarily saves uncommitted changes. Use when switching branches with uncommitted work or pulling updates.

3. What is the difference between git reset and git revert?

Answer: Reset moves HEAD to previous commit, potentially losing history. Revert creates new commit that undoes changes, preserving history.

4. What is a fast-forward merge?

Answer: When target branch hasn't diverged, Git simply moves the pointer forward instead of creating merge commit. Use --no-ff to force merge commit.

5. How do you resolve merge conflicts?

Answer: Edit conflicted files, remove conflict markers, add resolved files with git add, then complete merge with git commit.

6. What is the difference between git fetch and git pull?

Answer: Fetch downloads changes without merging. Pull is fetch + merge in one command.

7. How do you undo the last commit?

Answer: - Keep changes: git reset --soft HEAD~1 - Unstage changes: git reset HEAD~1 - Discard changes: git reset --hard HEAD~1
