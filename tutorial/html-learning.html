What is HTML?

Definition: HTML (HyperText Markup Language) is the standard markup language for creating web pages and web applications. It describes the structure and content of web documents using elements and tags.

Key Features:

  • Markup Language: Uses tags to define elements and structure
  • Platform Independent: Works across all browsers and devices
  • Easy to Learn: Simple syntax with readable tags
  • Semantic: Provides meaning to content structure
  • Extensible: Can be enhanced with CSS and JavaScript

HTML Evolution:

HTML 1.0 (1993) ‚Üí Basic structure and links
HTML 2.0 (1995) ‚Üí Forms and tables
HTML 3.2 (1997) ‚Üí Style sheets and scripts
HTML 4.01 (1999) ‚Üí Improved accessibility
XHTML 1.0 (2000) ‚Üí XML-based HTML
HTML5 (2014) ‚Üí Modern web standards

Basic HTML Document:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My First Web Page</title>
</head>
<body>
    <h1>Welcome to HTML</h1>
    <p>This is my first HTML document.</p>
</body>
</html>

HTML Document Anatomy

<!DOCTYPE html>  <!-- Document type declaration -->
<html lang="en">    <!-- Root element with language -->
<head>
    <!-- Meta information about the document -->
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <meta name="description" content="Page description for SEO">
    <meta name="keywords" content="HTML, CSS, JavaScript">
    <meta name="author" content="Your Name">
    
    <title>Page Title - Appears in Browser Tab</title>
    
    <!-- External resources -->
    <link rel="stylesheet" href="styles.css">
    <link rel="icon" href="favicon.ico">
    
    <!-- Internal styles and scripts -->
    <style>
        body { font-family: Arial, sans-serif; }
    </style>
</head>
<body>
    <!-- Visible content goes here -->
    <header>
        <h1>Website Header</h1>
        <nav>Navigation Menu</nav>
    </header>
    
    <main>
        <section>
            <h2>Main Content Section</h2>
            <p>Page content goes here.</p>
        </section>
    </main>
    
    <footer>
        <p>© 2024 Your Website</p>
    </footer>
    
    <!-- Scripts at the end for better performance -->
    <script src="script.js"></script>
</body>
</html>

Block vs Inline Elements

<!-- Block Elements (take full width, start on new line) -->
<div>Division - generic block container</div>
<p>Paragraph - text block</p>
<h1>Heading - title block</h1>
<section>Section - content section</section>
<article>Article - standalone content</article>
<header>Header - page/section header</header>
<footer>Footer - page/section footer</footer>
<nav>Navigation - navigation links</nav>

<!-- Inline Elements (take only needed width, stay in line) -->
<span>Span - generic inline container</span>
<a href="#">Link - hyperlink</a>
<strong>Strong - important text</strong>
<em>Emphasis - emphasized text</em>
<code>Code - inline code</code>
<img src="image.jpg" alt="Image"> <!-- Self-closing -->

Images

<!-- Image with different sources for different screen sizes -->
<picture>
    <source media="(min-width: 800px)" srcset="large-image.jpg">
    <source media="(min-width: 400px)" srcset="medium-image.jpg">
    <img src="small-image.jpg" alt="Responsive image">
</picture>

<!-- Image with multiple resolutions -->
<img src="image.jpg" 
     srcset="image-320w.jpg 320w, image-640w.jpg 640w, image-1280w.jpg 1280w"
     sizes="(max-width: 320px) 280px, (max-width: 640px) 600px, 1200px"
     alt="High-resolution image">

<!-- Clickable image -->
<a href="large-image.jpg">
    <img src="thumbnail.jpg" alt="Click to view larger image">
</a>

<!-- Figure with caption -->
<figure>
    <img src="chart.jpg" alt="Sales chart for 2024">
    <figcaption>Sales performance chart showing growth in Q3 2024</figcaption>
</figure>

Frequently Asked Questions

Q1: What is the difference between HTML and XHTML?

Answer:

  • HTML: More flexible syntax, case-insensitive tags
  • XHTML: XML-based, stricter syntax, case-sensitive
  • Closing Tags: XHTML requires all tags to be closed
  • Attributes: XHTML requires quoted attribute values

Q2: What are semantic HTML elements?

Answer: Semantic elements clearly describe their meaning to both browser and developer: header, nav, main, section, article, aside, footer, figure, time, mark.

Q3: What is the difference between block and inline elements?

Answer:

  • Block: Take full width, start on new line (div, p, h1-h6)
  • Inline: Take only needed width, stay in line (span, a, strong)
  • Inline-block: Hybrid behavior, can set width/height

Q4: What is the purpose of the DOCTYPE declaration?

Answer: DOCTYPE tells the browser which version of HTML the page is written in, ensuring proper rendering and standards mode activation.

Q5: How do you make a website responsive using HTML?

Answer: Use viewport meta tag, responsive images with srcset, flexible layouts with semantic elements, and CSS media queries.
